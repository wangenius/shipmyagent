## Code Review Automation

Automate pull request reviews with consistent, thorough feedback.

### Use Case

**Problem**: Manual code reviews are time-consuming and inconsistent.

**Solution**: ShipMyAgent reviews every PR automatically, catching issues before human review.

### Implementation

#### Step 1: Configure PR Review Agent

**ship.json**:

```json
{
  "integrations": {
    "github": {
      "enabled": true,
      "token": "${GITHUB_TOKEN}",
      "repository": "owner/repo"
    }
  }
}
```

#### Step 2: Create Review Task

**`.ship/task/pr-review/task.md`**:

```markdown
---
title: Automated PR Review
cron: "*/30 * * * *"
description: Review open PRs and post a summary
chatKey: telegram-chat-123456789
status: enabled
---

Review all open pull requests:

## Review Criteria

1. **Code Quality**
   - Follows project style guide
   - No obvious bugs
   - Proper error handling
   - Performance considerations

2. **Testing**
   - New tests included
   - Edge cases covered
   - Tests actually test the code

3. **Documentation**
   - API changes documented
   - Examples updated
   - Breaking changes noted
```

#### Step 3: Define Review Standards

**Agent.md** (add section):

```markdown
## Code Review Standards

### Must Have
- All new code is tested
- No console.log() in production code
- Error handling for async operations
- TypeScript strict mode compliance

### Nice to Have

- Documentation for complex logic

- Missing tests for critical paths
```

```
#### 2. Exploratory Analysis

```
You: "Analyze the performance bottleneck in the dashboard"

Agent:
[Analyzing dashboard code...]

**Bottleneck Analysis**

**Issue 1: N+1 Query Problem**
Location: src/dashboard/fetchers.ts:15

```typescript
// Loads user data for each dashboard item separately
items.forEach(item => {
  item.user = await getUser(item.userId);
});
```

Impact: For 50 items, makes 50 database calls.

**Fix**: Batch loading
```typescript
const userIds = items.map(i => i.userId);
const users = await getUsers(userIds);
items.forEach(item => {
  item.user = users.get(item.userId);
});
```

Reduces 50 queries to 1.

**Issue 2: Large Bundle Size**
- Current: 2.3MB
- Culprit: lodash (70KB), moment.js (67KB)

**Suggestion**: Replace with
  - lodash-es (tree-shakeable)
  - date-fns (smaller)

Estimated improvement: 35% smaller bundle

Implement these fixes?

```
#### 3. Code Generation
```

Here's a complete REST controller:

```typescript
// src/controllers/products.ts
import { Request, Response } from 'express';
import { ProductService } from '../services/product.service';

export class ProductController {
  constructor(private service = new ProductService()) {}

  async list(req: Request, res: Response) {
    const { page = 1, limit = 10 } = req.query;
    const products = await this.service.list({
      page: Number(page),
      limit: Number(limit)
    });
    res.json(products);
  }

  async get(req: Request, res: Response) {
    const { id } = req.params;
    const product = await this.service.get(id);
    if (!product) {
      return res.status(404).json({ error: 'Not found' });
    }
    res.json(product);
  }

  async create(req: Request, res: Response) {
    const product = await this.service.create(req.body);
    res.status(201).json(product);
  }

  async update(req: Request, res: Response) {
    const { id } = req.params;
    const product = await this.service.update(id, req.body);
    res.json(product);
  }

  async delete(req: Request, res: Response) {
    const { id } = req.params;
    await this.service.delete(id);
    res.status(204).send();
  }
}
```
