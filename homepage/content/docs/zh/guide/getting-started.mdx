# 快速开始

## 1. 安装

首先，全局安装 `shipmyagent` 命令行工具。确保你的 Node.js 版本 >= 18。

```bash
npm install -g shipmyagent
```

## 2. 初始化项目

进入你想要部署 Agent 的代码仓库根目录，运行初始化命令：

```bash
cd /path/to/your/project
shipmyagent init
```

交互式引导将询问以下信息：

- **Agent 名称**：给你的 Agent 起个名字（默认为当前目录名）。
- **LLM 模型**：选择你想要使用的模型，支持 Claude Sonnet 4、Claude Haiku、Claude 3.5 Sonnet、GPT-4、GPT-4o、DeepSeek Chat 等，推荐 `Claude Sonnet 4` 以获得最佳体验。
- **聊天适配器（可多选）**：可选启用 Telegram / 飞书 / QQ。**仅选中的 adapters 会写入 `ship.json.services.chat.adapters`**。
- **QQ 沙箱环境**（仅选择 QQ 时）：是否使用沙箱环境进行测试。
- **Skills 安装**（可选）：可选安装推荐的技能插件，如 Vercel 最佳实践、Web 设计指南、浏览器自动化等。

初始化完成后，它会在你的目录中创建：

```
your-project/
├── Agent.md           # Agent 系统提示词
├── .env.example       # 环境变量模板（推荐提交）
├── .env               # 本地环境变量（不要提交）
├── ship.json          # 配置文件
├── .claude/           # Claude 兼容目录
│   └── skills/        # Skills 存放位置
└── .ship/             # Agent 数据目录
    ├── task/          # 定时任务
    ├── logs/          # 操作日志
    ├── .cache/        # 缓存数据
    ├── profile/       # 用户画像记忆
    │   ├── Primary.md
    │   └── other.md
    ├── data/          # 数据存储
    ├── chat/          # 聊天记录
    ├── public/        # 公共资源
    ├── config/        # 配置文件
    │   └── mcp.json   # MCP 配置
    ├── skills/        # 项目级技能
    ├── schema/        # JSON Schema (IDE 智能提示)
    │   ├── ship.schema.json
    │   └── mcp.schema.json
    └── .debug/        # 调试信息
```

> **提示**：Profile 记忆系统会保存用户偏好，MCP 配置支持 Model Context Protocol，Schema 文件提供 IDE 自动补全。

## 3. 配置 LLM

通过 `.env` 填入 API Key（推荐），或直接写入 `ship.json`（注意不要将密钥提交到 Git）。

```json
{
  "llm": {
    "provider": "anthropic",
    "model": "claude-3-5-sonnet-20241022",
    "baseUrl": "https://api.anthropic.com/v1",
    "apiKey": "sk-ant-..." // 替换为你的 API Key
  }
}
```

## 4. 启动 Agent

配置完成后，启动 Agent Server：

```bash
shipmyagent .
# `shipmyagent .`：前台启动（当前终端运行）
# `shipmyagent start [path]`：后台启动（终端关闭也保持运行）
```

你会看到类似以下的输出：

```
🚀 启动 ShipMyAgent: /path/to/project
=== ShipMyAgent 启动 ===
...
任务调度器已初始化
（可选）某个 Adapter 已启用（例如 Telegram）
```

现在，你的 Agent 已经在线并开始根据配置运行了！
如果启用了 Telegram，你可以直接向 Bot 发送消息："帮我检查一下最近修改的代码"。
