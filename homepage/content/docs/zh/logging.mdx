# 日志与审计数据

> ⚠️ **简化模式（2026-02-03）**：当前发布的 `shipmyagent` 包已暂时移除 Tasks/Runs 与审批；可依赖的审计数据主要是 `.ship/logs` 与 `.ship/chats`。

ShipMyAgent 的“可审计”主要由 `.ship/` 目录下的多类数据共同构成：系统日志、Agent 执行日志、对话日志。

## `.ship/` 关键目录

```text
.ship/
  logs/       # 系统 Logger + Agent Logger + LLM 请求/响应
  chats/      # 对话消息（jsonl，可归档）
  .cache/     # 运行缓存（如 ingress 去重）
  mcp/        # MCP 配置
```

## 统一日志：`Logger`

当前版本已统一为一套 `Logger`（系统层 + Agent 执行层共用）：

- 文件：`.ship/logs/<YYYY-MM-DD>.jsonl`（JSONL）
- 落盘策略：每条日志都会 append（包含系统事件、Agent 执行、LLM 请求/响应）

## LLM 请求日志开关（重要）

默认会记录向 LLM provider 发出的请求（通常包含 system + messages），落到 `.ship/logs/<date>.jsonl` 中。

可通过以下方式关闭：

- 在 `ship.json` 中设置 `llm.logMessages=false`

## 对话与审计怎么查？

- **对话内容**：`.ship/chats/<chatKey>.jsonl`（超过阈值会生成 `.archive-<n>.jsonl`）
- **执行/审计日志**：`.ship/logs/<YYYY-MM-DD>.jsonl`
