## 产品定位

### 我们是什么

**ShipMyAgent 是一个 Agent Runtime 基础设施**，它将代码仓库启动为一个拥有权限、任务、人格与对外接口的 AI Agent。

### 核心差异化

| 维度 | GitHub Copilot | ChatGPT/Claude | **ShipMyAgent** |
|------|---------------|----------------|-----------------|
| **定位** | 代码补全工具 | 对话助手 | **Agent Runtime** |
| **核心能力** | 建议代码 | 回答问题 | **可执行、可审计的 AI** |
| **控制权** | ❌ 你被动接受 | ❌ 你被动接受 | ✅ **Human-in-the-Loop** |
| **可审计性** | ❌ 无记录 | ❌ 会话结束即丢 | ✅ **100% 可追溯** |
| **权限控制** | ❌ 无权限模型 | ❌ 无权限模型 | ✅ **最小权限 + 细粒度配置** |
| **持久化** | ❌ 会话级 | ❌ 会话级 | ✅ **项目级长期记忆** |

### 不是什么

- ❌ 不是"帮你写代码"的工具
- ❌ 不是企业 SaaS（v1）
- ❌ 不是 ChatGPT 替代品
- ❌ 不是"无代码"平台

### 是什么

- ✅ **开发者工具**：面向工程师的 Agent Runtime
- ✅ **基础设施**：Repo is the Agent
- ✅ **可控运行时**：AI ≠ Magic，AI = Controlled Runtime

## 目标用户

### 核心用户（v1）

#### 1. 高级程序员 / 开源维护者

**痛点：**
- 维护大型开源项目，需要 24/7 的协作者
- PR review 占用大量时间
- 问题追踪与维护工作繁重

**ShipMyAgent 价值：**
- Agent 成为项目的第 N+1 个成员
- 自动代码审查、建议优化
- 定期扫描 TODO、整理 issue

#### 2. Solo Founder / 小团队

**痛点：**
- 人手不足，无法 7×24 监控项目
- 运维、测试、代码审查占用核心开发时间
- 知识沉淀困难，新人上手慢

**ShipMyAgent 价值：**
- 自动运维任务（依赖更新、测试运行）
- "活的文档"：项目知识即 Agent 上下文
- 降低协作门槛

#### 3. AI-Native 开发者

**痛点：**
- 相信 AI 的潜力，但担心不可控
- 需要 AI 能够执行操作，而不只是建议
- 重视安全性与可审计性

**ShipMyAgent 价值：**
- Human-in-the-Loop 确保可控
- 完整的审计日志
- 声明式任务定义

#### 4. DevOps / Infra 工具链工程师

**痛点：**
- 内部工具众多，维护成本高
- 需要自动化运维，但担心脚本失控
- 审计与合规需求

**ShipMyAgent 价值：**
- 权限控制确保安全
- 所有操作可审计
- 自然语言接口降低使用门槛

### 非目标用户（v1）

- ❌ 非技术用户
- ❌ 纯内容创作者
- ❌ 大型企业 IT 部门（v1 不考虑私有化部署）

## v1 策略：开源优先

### 为什么 v1 必须开源？

1. **建立开发者心智**
   - ShipMyAgent = 安全、可控、可审计的 Agent Runtime
   - 开源 = 信任 = 基础设施的标准

2. **降低试用门槛**
   - `npm install -g shipmyagent` 立即体验
   - 无需注册、无需付费、无需审批

3. **社区驱动的产品迭代**
   - GitHub Issues = 真实需求
   - PR = 社区贡献
   - 成功案例 = 最佳实践

4. **技术壁垒的形成**
   - Permission-as-Code 标准

   - Agent Runtime 规范

## 潜在的商业模式

> **重要：v1 不追求商业化。以下只是未来的可能性。**

### 方向 1: Cloud 托管（Freemium）

**时机：** v2（12-18 个月后，社区成熟后）

**定位：** 解决"不想 24h 开机运行 Agent"的痛点

```yaml
定价:
  Free:
    - 本地运行（永久免费）
    - 完整功能
    - 社区支持

  Pro ($29/月):
    - 云端托管（3 个 Agent）
    - 30 天历史记录
    - 优先支持

  Team ($99/月):
    - 云端托管（10 个 Agent）
    - 无限历史
    - 团队协作
```

**商业模式：**
- 不是主要收入来源
- 而是"方便开发者"的增值服务
- 保持开源核心完全可用

---

### 方向 2: 企业私有化部署

**时机：** v2-v3（有明确的客户需求后）

**定位：** 服务于有安全/合规需求的企业

```yaml
定价: $25k-$100k/年

功能:
  - 私有化部署（Docker/Kubernetes）
  - SSO/SAML 集成
  - 审计日志导出
  - 企业级支持

触发条件:
  - 有企业主动咨询私有化部署
  - 有明确的合规需求（SOC2/ISO27001）
  - 开源版本已经验证价值
```

**关键原则：**
- **不强推**：等客户主动上门
- **不分散精力**：核心永远是开源 Runtime
- **Land & Expand**：先用免费版验证价值，再卖企业版

---

### 方向 3: Agent Marketplace（长期）

**时机：** v3（24 个月后）

**定位：** Agent 技能市场

```
ShipMyAgent Marketplace：
├─ 官方 Agent
│   ├─ React Maintainer ($199/年)
│   ├─ Python Data Scientist ($299/年)
│   └─ DevOps Automation ($399/年)
├─ 社区 Agent
│   └─ 第三方开发者发布
└─ 企业定制 Agent
    └─ 企业内部开发，不公开

定价：
  - 免费社区 Agent
  - 付费官方 Agent
  - Marketplace 抽成 30%
```

**前提条件：**
- ShipMyAgent 成为标准
- 有大量活跃用户
- 有成熟的 Agent 开发生态

## 关键原则

### DO ✅

1. **开源优先** - 建立开发者心智
2. **技术驱动** - 用技术壁垒构建护城河
3. **社区第一** - 用户反馈 > 商业利益
4. **长期主义** - 做"难但正确"的事
5. **保持专注** - v1 只做 Agent Runtime

### DON'T ❌

1. **不追求短期变现** - v1 不考虑付费
2. **不分散精力** - 不做"企业版"（v1）
3. **不做营销驱动** - 技术博客 > 广告投放
4. **不做伪需求** - 等用户主动提需求
5. **不做平台** - v1 只做 Runtime
